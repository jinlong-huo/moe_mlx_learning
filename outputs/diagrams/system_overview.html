2. 传输层对比: EPS 电交换 vs OCS 光交换

⚡ EPS (传统电分组交换)

连接模型始终在线，逐包路由
建立开销0（统计复用）
每跳延迟高（交换芯片排队+转发）
并发容量理论上无上限
典型延迟100–500 µs / 跳
适合场景流量不可预测、连接数多变
⚠ 每次 all-to-all 都付出完整的逐跳延迟
delay = latency + bytes / BW

🔬 OCS (光电路交换)

连接模型有限电路池，按需建立
建立开销冷启动: reconfig_time µs
热路径延迟极低（纯光学直通）
并发容量max_circuits 条电路
典型延迟1–2 µs / 跳（热路径）
适合场景流量模式稳定、可预测
✅ 热路径: delay = circuit_latency + bytes / BW
❌ 冷路径: 外加 reconfig_time（MEMS 微镜切换）
